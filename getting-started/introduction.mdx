# Introduction to Wacht

Wacht provides a robust set of primitives to handle everything from user authentication to complex B2B multi-tenancy and autonomous AI agents. We abstract away the heavy lifting of security, session management, and backend infrastructure so you can focus on building your core product.

## Why Wacht?

Modern applications require more than just simple login screens. They need nested organizations, secure API key provisioning for developers, event-driven webhooks, and increasingly, embedded AI capabilities. 

Wacht is built to solve these challenges out-of-the-box:

- **Comprehensive Auth**: Passwordless flows, seamless OAuth providers, waitlist management, and highly-secure managed sessions.
- **Deep Multi-Tenancy**: Native support for Organizations and Workspaces, enabling complex enterprise hierarchies and fine-grained role-based access control (RBAC).
- **Embedded AI**: Natively execute intelligent agents using our managed AI primitives, directly tied to user and organization storage contexts.
- **Type-safe SDKs**: Fully typed, production-ready SDKs across Next.js, React Router, TanStack Router, and Rust.

## Architecture Overview

Wacht operates on a few core concepts:

1. **Deployments**: Your standalone environment (e.g., Development, Staging, Production). Each deployment contains its own isolated database of users, organizations, and configurations.
2. **Users & Sessions**: Individual identities authenticated via your frontend integration. Wacht issues and verifies JWT sessions natively.
3. **Organizations & Workspaces**: The multi-tenant hierarchy. Users can be members of organizations and operate simultaneously within scoped sub-workspaces.
4. **Agents & Knowledge Bases**: AI entities that can act autonomously on behalf of users or organizations, drawing from your uploaded vectorized knowledge bases.
