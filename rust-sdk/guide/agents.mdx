# AI API Guide

The Rust SDK targets the backend API surface.

That surface now has two layers:

1. AI control-plane APIs for deployment-owned resources.
2. Actor-project runtime administration APIs for backend operators.

The older execution-context and integration APIs are removed.

## Control plane

Use the control-plane clients for deployment-owned AI resources:

- agents
- agent details and sub-agents
- tools
- knowledge bases and documents
- MCP servers
- agent-to-tool attachments
- agent-to-knowledge-base attachments
- AI settings

```rust
use wacht::{WachtClient, WachtConfig};

let client = WachtClient::new(WachtConfig::new("wk_live_...", "https://app.wacht.dev"))?;

let agents = client.ai().agents().list_agents().send().await?;
let tools = client.ai().tools().list_tools().send().await?;
let knowledge_bases = client.ai().knowledge_bases().fetch_knowledge_bases().send().await?;
let settings = client.ai_settings().fetch_ai_settings().send().await?;
```

## Actor-project runtime administration

Use the runtime-admin clients for backend-side actor-scoped resources:

- actor projects
- project boards and board items
- actor project threads
- thread events, assignments, task graphs, messages, filesystem
- actor MCP server connections

```rust
use wacht::{WachtClient, WachtConfig};

let client = WachtClient::new(WachtConfig::new("wk_live_...", "https://app.wacht.dev"))?;

let projects = client
    .ai()
    .actor_projects()
    .list_actor_projects("123")
    .send()
    .await?;

let threads = client
    .ai()
    .actor_project_threads()
    .search_threads("123")
    .query("review")
    .send()
    .await?;

let actor_mcp = client
    .ai()
    .actor_mcp_servers()
    .list_actor_mcp_servers("123")
    .send()
    .await?;
```

## Current backend contract

The backend runtime surface centers on:

- `actor-projects`
- `actor-project-threads`
- `actor-mcp-servers`

It does not expose session-scoped frontend runtime APIs such as:

- `/ai/session`
- `/ai/projects/*`
- `/ai/threads/*`
- `/ai/threads/:thread_id/stream`
- `/notifications/stream`

## Recommendation

Use the Rust SDK for trusted server-side administration.

Use the frontend API and React SDK for browser/session-scoped runtime behavior and streaming.
