The `wacht` crate provides Rust access to Wacht APIs and auth primitives.

## Main use cases

- **Typed API calls** with `WachtClient`
- **Gateway authz checks** for API key and OAuth access token principals
- **Axum middleware integration** for JWT auth and permission checks

## Typical usage pattern

1. Build a client from env/config.
2. Use client-scoped APIs (`client.users()`, `client.webhooks()`, etc.).
3. Use `client.gateway()` when validating machine principals.
4. Add Axum auth layers when serving HTTP routes.
