---
title: 'Environment Variables'
description: 'Reference for wacht-rs runtime configuration.'
---

## Required

<ParamField path="WACHT_API_KEY" type="string" required>
API key used for backend API authentication.
</ParamField>

<ParamField path="WACHT_FRONTEND_HOST" type="string">
Frontend host used for session/JWK related flows.
Optional when `WACHT_PUBLISHABLE_KEY` is provided.
</ParamField>

<ParamField path="WACHT_PUBLISHABLE_KEY" type="string">
Publishable key (`pk_test_...` or `pk_live_...`).
When set, SDK derives frontend host automatically.
</ParamField>

## Optional

<ParamField path="WACHT_PUBLIC_SIGNING_KEY" type="string">
PEM public key override for token validation.
</ParamField>

## Resolution order for frontend host

1. `WACHT_FRONTEND_HOST` (if present)
2. Derived from `WACHT_PUBLISHABLE_KEY`
3. Error if neither exists

## Example

```bash
WACHT_API_KEY=wk_live_xxx
WACHT_PUBLISHABLE_KEY=pk_test_xxx
# optional
WACHT_PUBLIC_SIGNING_KEY="-----BEGIN PUBLIC KEY-----..."
```
