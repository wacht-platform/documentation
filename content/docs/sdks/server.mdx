---
title: Server SDKs
description: Where backend integrations belong once the frontend SDK has established user and deployment context.
---

Server-side integrations fall into two broad buckets:

- request authentication and token validation
- management and automation against Wacht APIs

## Node

The Node package is the natural fit for JavaScript and TypeScript backends that need to operate on users, organizations, webhooks, API auth, and AI-related admin surfaces.

## Rust

The Rust SDK fits services that need stronger type guarantees or are already part of the Rust platform stack.

## Recommended docs structure

- one getting-started page per runtime
- one auth section
- one management/API section
- generated type and contract reference where automation is practical
