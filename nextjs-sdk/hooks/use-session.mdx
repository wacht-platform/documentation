## Purpose

`useSession()` is the shared frontend session hook used by the Wacht React SDK packages.

It is implemented once in the shared JSX layer and re-exported by the framework packages.

This hook is responsible for:

- loading the current frontend session from `GET /session`
- switching the active sign-in inside the current browser session
- signing out the current session or one sign-in inside it
- switching the active organization and workspace context
- exchanging one-time session tickets
- minting JWTs from `GET /session/token`
- clearing cached JWTs when session state changes

## Import

```tsx
import { useSession } from '@wacht/nextjs'
```

## Current return shape

`useSession()` returns a discriminated union with two states.

### Loading state

```ts
{
  loading: true
  error: Error | null
  refetch: () => Promise<void>
}
```

In the loading state, the mutation helpers and `session` are intentionally unavailable.

### Ready state

```ts
{
  loading: false
  error: Error | null
  session: Session
  switchSignIn: (signInId: string) => Promise<void>
  signOut: (signInId?: string) => Promise<void>
  getToken: (template?: string) => Promise<string>
  switchOrganization: (organizationId?: string) => Promise<void>
  switchWorkspace: (workspaceId: string) => Promise<void>
  exchangeTicket: (ticket: string) => Promise<void>
  refetch: () => Promise<void>
}
```

## Returned fields

<ResponseField name="loading" type="boolean">
  `true` while the hook is still resolving the current session through SWR or while the underlying client is not ready.
</ResponseField>

<ResponseField name="error" type="Error | null">
  The current SWR error for session loading.
</ResponseField>

<ResponseField name="session" type="Session">
  The hydrated frontend session returned by `GET /session`.
</ResponseField>

<ResponseField name="refetch" type="() => Promise<void>">
  Revalidates the current session by forcing the session SWR key to re-fetch.
</ResponseField>

<ResponseField name="getToken" type="(template?: string) => Promise<string>">
  Calls `GET /session/token` and returns the signed JWT string.
</ResponseField>

<ResponseField name="switchSignIn" type="(signInId: string) => Promise<void>">
  Calls `POST /session/switch-sign-in?sign_in_id=...`, clears the token cache, refreshes the session, and revalidates organization and workspace memberships.
</ResponseField>

<ResponseField name="signOut" type="(signInId?: string) => Promise<void>">
  Calls `POST /session/sign-out` or `POST /session/sign-out?sign_in_id=...`, clears cached session-related data, clears the token cache, refreshes the session, and applies configured sign-out redirects from deployment UI settings when present.
</ResponseField>

<ResponseField name="switchOrganization" type="(organizationId?: string) => Promise<void>">
  Calls `POST /session/switch-organization`, clears the token cache, and refreshes the session.
</ResponseField>

<ResponseField name="switchWorkspace" type="(workspaceId: string) => Promise<void>">
  Calls `POST /session/switch-workspace?workspace_id=...`, clears the token cache, and refreshes the session.
</ResponseField>

<ResponseField name="exchangeTicket" type="(ticket: string) => Promise<void>">
  Calls `GET /session/ticket/exchange?ticket=...`, clears the token cache, and refreshes the session.
</ResponseField>

## Route contract used by this hook

The current implementation calls these frontend routes:

- `GET /session`
- `GET /session/token`
- `POST /session/switch-sign-in?sign_in_id=...`
- `POST /session/sign-out`
- `POST /session/sign-out?sign_in_id=...`
- `POST /session/switch-organization?organization_id=...`
- `POST /session/switch-workspace?workspace_id=...`
- `GET /session/ticket/exchange?ticket=...`

Reference:

- [Frontend Session And Auth Routes](/guides/authentication/frontend-session-auth-routes)

## `getToken(template?)`

The hook keeps a per-template in-memory token cache.

Current behavior:

- default template name is `default`
- cached tokens are reused until `expires > Date.now()` is no longer true
- concurrent token fetches for the same template are de-duplicated with a singleton promise map
- the token cache is cleared after sign-in context changes such as sign-out, sign-in switching, organization switching, workspace switching, or ticket exchange

Example:

```tsx
import { useSession } from '@wacht/nextjs'

export function UseTokenButton() {
  const { loading, session, getToken } = useSession()

  if (loading || !session) return null

  return (
    <button
      onClick={async () => {
        const token = await getToken('default')
        console.log(token)
      }}
    >
      Get token
    </button>
  )
}
```

## Example

```tsx
import { useSession } from '@wacht/nextjs'

export function SessionPanel() {
  const {
    loading,
    error,
    session,
    getToken,
    switchOrganization,
    switchWorkspace,
    signOut,
  } = useSession()

  if (loading) return <div>Loading session...</div>
  if (error) return <div>Failed: {error.message}</div>

  return (
    <div>
      <p>Session ID: {session.id}</p>
      <p>Active sign-in: {session.active_signin?.id ?? 'none'}</p>
      <button onClick={() => getToken()}>Mint JWT</button>
      <button onClick={() => switchOrganization()}>Clear organization</button>
      <button onClick={() => switchWorkspace('123')}>Switch workspace</button>
      <button onClick={() => signOut()}>Sign out all sign-ins</button>
    </div>
  )
}
```

## Behavior notes

- the hook refreshes the session every 30 seconds through SWR
- revalidation on focus and reconnect is disabled in the current implementation
- SWR deduping interval is 5 seconds
- when the session is not ready, the hook returns the loading branch instead of `Session | null`
- membership caches are explicitly revalidated after `switchSignIn(...)`
- sign-out clears multiple cached keys, including notification and AI session keys
