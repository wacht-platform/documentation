## Environment-driven config

`WachtConfig::from_env()` requires:

- `WACHT_API_KEY`
- one of:
  - `WACHT_FRONTEND_HOST`
  - `WACHT_PUBLISHABLE_KEY` (derived automatically)

Optional:

- `WACHT_PUBLIC_SIGNING_KEY`

## Programmatic config

```rust
use wacht::{WachtClient, WachtConfig};

let config = WachtConfig::new(
    "wk_live_xxx",
    "https://your-deployment.fapi.trywacht.xyz",
);

let client = WachtClient::new(config)?;
```

## Environment config example

```rust
use wacht::WachtClient;

let client = WachtClient::from_env().await?;
```

## Public key behavior

- If `WACHT_PUBLIC_SIGNING_KEY` is present, it is used directly.
- Otherwise `load_public_key()` can fetch from `/.well-known/jwk` on the frontend host.

```rust
use wacht::WachtConfig;

let config = WachtConfig::from_env()?.load_public_key().await?;
```

## Legacy global init (optional)

```rust
use wacht::{init, init_from_env};

init_from_env().await?;
// or
init(config)?;
```
