# AI API

`@wacht/backend` now exposes two distinct AI surfaces:

1. Control-plane APIs for deployment resources.
2. Actor-project runtime administration APIs for backend operators.

Use this SDK from trusted server environments only.

```ts
import { WachtClient } from "@wacht/backend";

const client = new WachtClient({ apiKey: process.env.WACHT_API_KEY! });
```

## Scope split

## Example: create an actor project

```ts
const project = await client.ai.createActorProject("123", {
  name: "Website Launch",
  agent_id: "456",
  description: "Coordinate launch preparation and review.",
});
```

## Example: create a project thread

```ts
const thread = await client.ai.createAgentThread(project.id, {
  title: "Launch coordinator",
  agent_id: "456",
  thread_purpose: "coordinator",
  accepts_assignments: true,
});
```

## Example: append task journal

```ts
await client.ai.appendProjectTaskBoardItemJournal(project.id, "789", {
  summary: "Reviewed copy and requested revisions",
  body_markdown: "Headline is ready. CTA still needs legal review.",
});
```

## Example: run a thread

```ts
const execution = await client.ai.runAgentThread(thread.id, {
  execution_type: {
    new_message: {
      message: "Review the launch plan and produce next actions.",
    },
  },
});
```

## Important contract changes

The current SDK no longer models these removed backend concepts:

- actor CRUD routes
- `/ai/projects/*` runtime routes on the backend API
- `/ai/threads/:thread_id/state`
- `/ai/threads/:thread_id/status-updates`
- `/ai/thread-events/:event_id`
- `/ai/thread-task-graphs/:graph_id/*`

Backend runtime administration now uses:

- `/ai/actor-projects/*`
- `/ai/actor-project-threads/*`
- `/ai/actor-mcp-servers/*`

For end-user runtime access and streaming, use the frontend API and React SDK instead.
